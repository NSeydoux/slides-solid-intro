Why would you want to build a Solid app?

Pain points building a new app

    For the developer

  • Onboarding is tedious
  • Competing with established apps
  • Liability for data

    For the user

  • Accounts proliferation
  • Data overlap

Components of the Solid architecture

    The user

  • They control the data.
  • They use apps to get services.

    The app

  • It consumes the data.
  • It creates more data.

    The Pod

  • It hosts the data.
  • It enforces access permissions.

    The Identity Provider

  • IdP for short.
  • It identifies the user.

Logging in a Solid app

    Profile discovery

  • The user provide their WebID to the app.

    IdP discovery

  • The app looks up the user's profile
  • The profile links to the identity provider.

    The OIDC dance

  • The user logs in the identity provider via OIDC

    OIDC outcome

  • The app gets an access and an ID token.
  • The ID token contains the WebID

    Authenticated access

  • The app can access private resources.
